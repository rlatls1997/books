## 2.2 여기저기에서 쓰이는 이진 탐색(Binary Search)
1부터 N사이의 숫자를 탐색한다면 log<sub>2</sub>N 번의 추측 내에 맞출 수 있다.  
시간복잡도 : O(logN)

<img src="https://github.com/user-attachments/assets/c1849000-bbb1-4528-805e-709519486e22" width="600">

순차탐색은 평균적으로 n/2번 비교하는 반면 이진탐색은 최대 log<sub>2</sub>N 번 비교한다 

이진 탐색의 응용은 정렬된 목록을 빠르게 탐색하는 것에서 그치지 않는다.  
입력 파일을 읽고 어느 잘못된 줄을 제거하기 위해 사용할 수도 있다.

A 문제에 대해 생각해보자.
<br><br>

#### 만약 메모리가 넉넉하다면?
이전 장에서 봤던 비트맵 기법을 사용해서 이미 확인된 정수를 표시할 수 있다.

확인된 정수를 표현하기 위해 2<sup>32</sup> 비트(536,870,912바이트, 512MB)가 필요하다.
<br><br>

#### 수백 바이트 정도의 메모리와 서너 개의 임시 파일만 쓸 수 있다면?
메모리가 제한되어 비트맵 기법은 사용할 수 없다.

이진탐색을 적용하기 위해서는
- 어떤 범위와 그 범위 안에 포함되는 원소들에 대한 표현을 정의해야함 => 이 문제에서 범위는 
- 주어진 범위의 양쪽 절반 중 어느쪽에 존재하지 않는 정수가 포함되는지 알애내는 검사방법 필요 => 예상 개수보다 적은 원소를 가진 쪽 

과정을 예로 들면,
40억의 절반인 20억 번째의 정수가 target[2,000,000,000] = 3,000,000,000 라고 가정하면,  
1~20억 번째에는 1~30억 사이의 정수가 있고, 20억+1~40억 번째에는 30억+1~42억9천만 사이의 정수가 있으므로  
1~20억 사이에는 존재하지 않는 정수가 반드시 존재한다.

```java
...
    /**
     * 이진 탐색으로 누락된 숫자를 찾는 메소드
     * @param arr 정렬된 정수 배열
     * @param startValue 시작 숫자 (여기서는 1)
     * @return 찾은 첫 번째 누락된 숫자
     */
    public static int findFirstMissing(int[] arr, int startValue) {
        int low = 0;
        int high = arr.length - 1;

        // 만약 첫 번째 숫자부터 1(startValue)이 아니라면 1이 누락된 것
        if (arr[low] != startValue) {
            return startValue;
        }

        while (low <= high) {
            int mid = low + (high - low) / 2;

            // arr[mid]가 있어야 할 값(mid + startValue)보다 크다면
            // 왼쪽 구간 어딘가에 빈틈이 있다는 뜻입니다.
            if (arr[mid] > (mid + startValue)) {
                high = mid - 1;
            } 
            // 값이 일치한다면 왼쪽은 빈틈이 없으므로 오른쪽 구간을 탐색합니다.
            else {
                low = mid + 1;
            }
        }

        // 탐색이 끝난 시점의 low 인덱스에 startValue를 더한 값이 첫 번째 누락된 숫자입니다.
        return low + startValue;
    }
...
```

예를 들어, 정렬된 배열이 `[1, 2, 3, 5, 6, 7, 8, 9]` 라면
- 첫 번째 루프에서 : low=0, high=7, mid=3 → arr[3]=5, 3+1=4 → arr[3] > 4 이므로 high=2
- 두 번째 루프에서 : low=0, high=2, mid=1 → arr[1]=2, 1+1=2 → arr[1] == 2 이므로 low=2
- 세 번째 루프에서 : low=2, high=2, mid=2 → arr[2]=3, 2+1=3 → arr[2] == 3 이므로 low=3  

탐색 종료 후 low=3 이므로 3+1=4 가 첫 번째 누락된 숫자이다.

과정을 보면,
- 첫 번째 패스 : 전체 파일을 읽어 32번째 비트가 0인 것과 1인 것을 file0, file1에 각각 기록한다. 각 파일에 넣은 숫자의 카운트는 메모리상의 변수로 센다.
- 두 번째 패스 : 카운트가 적은 쪽의 파일을 읽어 31번째 비트가 0인 것과 1인 것을 file0, file1에 각각 기록한다.
- 세 번째 패스 : 카운트가 적은 쪽의 파일을 읽어 30번째 비트가 0인 것과 1인 것을 file0, file1에 각각 기록한다.
- ...
- 마지막 패스 : 1번째 비트에 대해 같은 작업을 수행한다.

각 단계를 작업하면서 해당 자리에 대한 비트를 저장한다. 예로..
- 32번째 비트가 0인 쪽이 적다면 [0XXXX...XXX]
- 31번째 비트가 1인 쪽이 적다면 [01XXX...XXX]
- 30번째 비트가 0인 쪽이 적다면 [010XX...XX]
- ...
- 1번째 비트가 1인 쪽이 적다면 [010...01]

이렇게 해서 최종적으로 존재하지 않는 정수를 찾을 수 있다.

기초적인 이진 탐색 기법이라도 약간의 응용으로 적절히 사용할 수 있다


## 2.3 기초적인 조작의 위력
문제 B를 살펴보자.
- n개의 원소를 가지는 벡터 x를 i만큼 왼쪽으로 회전시켜야 한다.
- 회전에 드는 시간은 n에 비례해야한다
- 수십 바이트 정도의 메모리만을 사용할 수 있다.

예를 들어 n=8, i=3 인 경우 `abcdefgh` → `defghabc`

여러가지 해법이 있을 수 있다.
<br><br><br>

첫 번째로,  
x의 처음 i개의 원소를 임시 배열에 복사하고, 나머지 n-i개의 원소를 i만큼 왼쪽으로 옮긴 후 x의 뒤쪽에 임시 배열에 있던 i개의 원소를 다시 복사하는 방법이 있다.

예를 들어 n=8, i=3 인 경우
-  처음 3개의 원소를 임시 배열에 복사 : 임시배열 `[a, b, c]`, x = `[a, b, c, d, e, f, g, h]`
-  나머지 5개의 원소를 왼쪽으로 3칸 이동 : 임시배열 `[a, b, c]`, x = `[d, e, f, g, h, d, e, f]`
-  임시 배열의 원소를 다시 복사 : x = `[d, e, f, g, h, a, b, c]`

이 방식은 i만큼의 메모리가 사용되므로 메모리 낭비이다.  
또한 x를 왼쪽으로 1씩 회전시키는 함수를 만들고 이를 i번 반복하는 방법도 있지만 이 경우에는 실행시간이 문제가 된다.
<br><br><br>

두 번째로
저글링 조작을 사용할 수 있다.

저글링 조작은, x[0]를 임시저장소인 t로 옮긴 후 x[i]를 x[0]으로 옮기고, x[2i]를 x[i]로 옮기고 ...   
이런식으로 계속 진행하다가 다시 x[0]으로 돌아오면 t를 그 위치에 넣는 방식이다

한 번의 조작으로 모든 원소가 옮겨지지 않았다면,  
다시 x[1]을 시작으로 조작을 가한다.

예를 들어 n=8, i=3 인 경우
-  x[0]를 t로 옮김 : t=`a`, x=`[a, b, c, d, e, f, g, h]`
-  x[3]를 x[0]으로 옮김 : t=`a`, x=`[d, b, c, d, e, f, g, h]`
-  x[6]를 x[3]으로 옮김 : t=`a`, x=`[d, b, c, g, e, f, g, h]`
-  x[1]를 x[6]으로 옮김 : t=`a`, x=`[d, b, c, g, e, f, b, h]`
- x[4]를 x[1]으로 옮김 : t=`a`, x=`[d, e, c, g, e, f, b, h]`
- x[7]를 x[4]로 옮김 : t=`a`, x=`[d, e, c, g, h, f, b, h]`
- x[2]를 x[7]으로 옮김 : t=`a`, x=`[d, e, c, g, h, f, b, c]`
- x[5]를 x[2]로 옮김 : t=`a`, x=`[d, e, f, g, h, f, b, c]`
- t를 x[5]로 옮김 : t=``, x=`[d, e, f, g, h, a, b, c]`

이 알고리즘은 각 요소를 정확히 한 번씩만 건드리기 때문에 O(n) 시간에 실행된다.
<br><br><br>

세 번째로  
관점을 바꾸는 것이다.

벡터 x를 회전시키는 것은 결국 벡터 ab의 두 부분을 바꿔 ba로 만드는 것과 같다. (여기서 a는 x의 처음 i개의 원소를 나타낸다.)  
b를 b<sub>l</sub>와 b<sub>r</sub>로 나누고 b<sub>r</sub>의 원소의 개수를 a의 원소의 개수와 같게 만든다.  

a와 b<sub>r</sub>를 교환하면, a의 이동은 완료된 것이므로 이제 b의 두 부분을 바꾸는 문제에만 집중 할 수 있다.

---
위 문제를 다시 생각해볼 수 있다.

결국 ab를 변환하여 ba로 만드는 문제이다.  


배열의 특정 부분의 원소들을 반전시키는 함수가 있다고 가정하면,  
ab에서 a를 반전한 a<sup>r</sup>b를 만들 수 있고,  
여기서 b를 반전한 a<sup>r</sup>b<sup>r</sup>를 만들 수 있다.

마지막으로 a<sup>r</sup>b<sup>r</sup>의 모든 원소를 반전시키면 ba가 된다.  
`a<sup>r</sup>b<sup>r</sup> == (ab)<sup>r</sup> == ba`

반전 메서드가 reverse라고 할 때 과정을 살펴보면..  
- reverse(0, i-1) : `[c, b, a, d, e, f, g, h]`
- reverse(i, n-1) : `[c, b, a, h, g, f, e, d]`
- reverse(0, n-1) : `[d, e, f, g, h, a, b, c]`


반전코드는 메모리 사용량이 적고 빠르며 짧고 간단하다.
- 시간복잡도 : O(n)
- 공간복잡도 : O(1)

한 줄에 한 단어씩 소문자로 230,000건이 기록된 영어단어 사전이 주어지고,  
이 중에서 모든 전철어구의 집합을 찾아내야 한다.

많은 접근방법이 있지만 비효율적이고 복잡한 방법이 대다수이다.  
<br><br>

먼저 어떤 단어에 포함된 모든 문자의 순열을 고려하는 방법이 있다.  
예로 'cholecystoduodenectomy' 라는 단어의 경우 22! (약 1.124 x 10<sup>21</sup>)개의 순열을 가진다.

순열을 만드는 시간이 1ps(10<sup>-12</sup>초),  
단어사전에서 전철어구인지 비교하기 위해 1μs(10<sup>-6</sup>초)가 걸린다고 가정하면,

단어사전에 있는 모든 단어의 전철어구를 찾는 시간만 따져봐도  
`230,000(개 단어) x 230,000(비교/단어) x 1μs = 52,900초 (약 14.7시간)` 이 걸린다.

위와 같은 문제를 피하기 위해 같은 전철어구끼리는 같은 단어로 표시할 수 있다.  
이렇게 표시해놓고 같은 표시를 가진 단어를 모으기만 하면 된다.

1. 같은 단어인지 확인하고 표시
2. 같은 표시의 단어 모으기


`deposit`, `dopiest`는 같은 문자로 구성되어 있으므로 정렬하면 `deiopst`가 된다.
1번 과정에서 정렬을 기반으로 만든 표시를 사용할 수 있다.

이후 같은 표시의 단어를 모아 전철어구를 출력할 수 있다.  
(2.8 절 에서 확인)



## 2.5 원리
#### 정렬
정렬 자체가 시스템의 요구사항 일 수도 있지만,  
다른 프로그램을 위한 준비 단계일 수도 있다. (위 전철어구 찾는 문제처럼, 관심사는 정렬이 아니지만 과정에 정렬이 필요할 수 있음)

#### 이진 탐색
이진탐색의 유일한 단점은 전체 집합을 미리 알아야 하고,  
이미 집합이 정렬된 상태여야 한다는 점이다.

#### 표시
동치관계가 집합을 정의하는 경우,  
그 집합에 포함되는 모든 원소가 같은 표시를 가지도록 정의하는 것은 도움이 된다.

단어 내의 문자를 정렬하여 하나의 전철어구 집합에 대한 표시를 할 수 있다.  
정렬을 한 후 같은 문자의 수를 나타내는 것도 표시가 될 수 있다.  
`Mississippi` -> `iiiimppssss` -> `i4m1p2s4` -> `i4mp2s4(1생략)`

또는 26개의 정수로 이루어진 배열(`[a, b, c, ... , x, y, z]` 각 철자에 맞게..)을 사용할 수도 있다

<br><br>
이런 표시는 사운덱스(Soundex, 발음이 비슷한 단어는 동일한 코드가 되도록 부호화 하는 알고리즘) 휴리스틱(어림짐작) 에서도 응용된다.

``` 발음 코드 표
- b, f, p, v -> 1
- c, g, j, k, q, s, x, z -> 2
- d, t -> 3
- l -> 4
- m, n -> 5
- r -> 6
```
위 표를 참고하여

- 첫 번째 문자는 그대로 복사
- 2번째 문자부터 위 발음 코드 표를 참고하여 변경
- 위 표에 없는 다른 문자는 무시
- 반복되는 문자는 한 번만 기록
- 최종 코드는 4자리로 맞춤 (모자르면 0으로 채움)

이 과정을 거쳐 만들어진다.

| 단어 | 발음 코드 |
|---|-------|
| Smith | S530  |
| Smythe | S530  |
| Schultz | S243  |
| Shultz | S432  |

#### 문제 정의
1장애서 사용자가 원하는 것이 무엇인지 파악하는 것이 필수적인 요소인 것을 봤다.

이 칼럼의 주제는 문제를 정의하고 나서, `어떤 기초적인 조작을 통해 문제를 해결한 것인가?` 이다.

#### 문제 해결자의 관점
처음의 아이디어를 바로 적용하려고 달려들기보다는

## 2.6 연습문제
### 연습문제 3. 두 가지 벡터 회전 알고리즘(저글링 조작, 반전코드)를 알고리즘으로 구현하라. <br>각 프로그램에서 i와 n의 최대공약수가 어떻게 사용되는가?
#### 1. 저글링 조작 알고리즘
```java
public class ArrayRotation {
    // n개 원소를 i만큼 왼쪽으로 회전시키는 저글링 알고리즘
    public void leftRotate(char[] arr, int i, int n) {
        // i가 n보다 클 경우를 대비
        i = i % n;
        if (i == 0) {
            return;
        }

        int gcdValue = gcd(i, n);

        for (int j = 0; j < gcdValue; j++) {
            // 한 세트의 시작 원소를 temp에 보관
            char temp = arr[j];
            int k = j;

            while (true) {
                // 다음에 가져올 원소의 위치 계산
                int next = (k + i) % n;

                // 다시 시작점으로 돌아왔다면 루프 종료
                if (next == j) {
                    break;
                }

                // 원소 이동
                arr[k] = arr[next];
                k = next;
            }
            // 보관했던 원소를 마지막 빈자리에 넣음
            arr[k] = temp;
        }
    }

    // 최대공약수(GCD) 계산 함수
    private int gcd(int a, int b) {
        if (b == 0) {
            return a;
        }
		
        return gcd(b, a % b);
    }

    public static void main(String[] args) {
        ArrayRotation ar = new ArrayRotation();
        char[] data = "abcdefgh".toCharArray();
        
        ar.leftRotate(data, 3, 8);
        System.out.println(new String(data)); // 결과: defghabc
    }
}
```

예를 들어 n=8, i=3 인 경우
-  x[0]를 t로 옮김 : t=`a`, x=`[a, b, c, d, e, f, g, h]`
-  x[3]를 x[0]으로 옮김 : t=`a`, x=`[d, b, c, d, e, f, g, h]`
-  x[6]를 x[3]으로 옮김 : t=`a`, x=`[d, b, c, g, e, f, g, h]`
-  x[1]를 x[6]으로 옮김 : t=`a`, x=`[d, b, c, g, e, f, b, h]`
- x[4]를 x[1]으로 옮김 : t=`a`, x=`[d, e, c, g, e, f, b, h]`
- x[7]를 x[4]로 옮김 : t=`a`, x=`[d, e, c, g, h, f, b, h]`
- x[2]를 x[7]으로 옮김 : t=`a`, x=`[d, e, c, g, h, f, b, c]`
- x[5]를 x[2]로 옮김 : t=`a`, x=`[d, e, f, g, h, f, b, c]`
- t를 x[5]로 옮김 : t=``, x=`[d, e, f, g, h, a, b, c]`

최대공약수는 `배열 내에서 만들어지는 독립적인 순환 고리의 개수` 이다.

위 예시를 순환을 그룹지어 나타내면
- cycle 1 : (0 → 3 → 6 → 1 → 4 → 7 → 2 → 5 → 0)

3과 8은 서로소이므로 사이클은 한 개만 생김


#### 2, 반전코드 알고리즘
```java
public class ArrayRotationReversal {
    public void leftRotate(char[] arr, int i, int n) {
        if (n == 0) return;
        
        // i가 n보다 클 경우를 대비한 나머지 연산
        i = i % n;
        if (i == 0) return;

        // 1단계: 앞부분 (0 ~ i-1) 반전
        reverse(arr, 0, i - 1);
        
        // 2단계: 뒷부분 (i ~ n-1) 반전
        reverse(arr, i, n - 1);
        
        // 3단계: 전체 반전 (0 ~ n-1)
        reverse(arr, 0, n - 1);
    }

    // 배열의 특정 구간을 반전시키는 helper 함수
    private void reverse(char[] arr, int start, int end) {
        while (start < end) {
            char temp = arr[start];
            arr[start] = arr[end];
            arr[end] = temp;
            start++;
            end--;
        }
    }

    public static void main(String[] args) {
        ArrayRotationReversal ar = new ArrayRotationReversal();
        char[] data = "abcdefgh".toCharArray();
        
        ar.leftRotate(data, 3, 8);
        System.out.println(new String(data)); // 출력: defghabc
    }
}
```

과정을 다시 살펴보면..
- reverse(arr, 0, 2) : `[c, b, a, d, e, f, g, h]`
- reverse(arr, 3, 7) : `[c, b, a, h, g, f, e, d]`
- reverse(arr, 0, 7) : `[d, e, f, g, h, a, b, c]`

최대 공약수가 필요하지 않음


|특징||저글링(Juggling)|반전(Reversal)|
|---|---|---|---|
|코드 복잡도||상대적으로 복잡 (GCD 필요)|매우 간단함|
|데이터 이동 횟수||각 원소당 정확히 1번 이동|각 원소당 2번 이동 (효율은 약간 낮음)|
|캐시 효율성||낮음 (인덱스를 건너뜀)|높음 (인접한 메모리 접근)|
|메모리 사용량||O(1) (수십 바이트)|O(1) (수십 바이트)|

## 2.8 전철어구 프로그램의 구현
세 단계의 파이프라인으로 구성된 전철어구 프로그램

<img src="https://github.com/user-attachments/assets/cbff873b-a66a-4ebd-8ce3-2296167cc57b" width="600">

- sign : 단어에 표시를 생성
- sort : 표시된 파일을 정렬
- squash : 전철어구가 되는 단어들을 한 줄로 합침

**sign 단계**
```java
import java.util.*;

public class Sign {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        while (sc.hasNext()) {
            String word = sc.next();
            char[] chars = word.toCharArray();
            Arrays.sort(chars);
            System.out.println(new String(chars) + " " + word);
        }
    }
}
```

sign단계에서는 파일의 `단어` 읽고 `표시`를 만들어서 `표시 단어` 형태로 출력한다.

**sort 단계**
```java
import java.util.*;

public class Sort {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        List<String> lines = new ArrayList<>();
		
        while (sc.hasNextLine()) {
            lines.add(sc.nextLine());
        }

        Collections.sort(lines);
        lines.forEach(System.out::println);
    }
}
```

sort 단계에선 정렬을 통해 같은 표시를 가지는 단어들을 모아준다.

**squash 단계**
```java
import java.util.*;

public class Squash {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        String lastKey = "", output = "";

        while (sc.hasNext()) {
            String key = sc.next();
            String word = sc.next();

            if (!key.equals(lastKey) && !lastKey.isEmpty()) {
                System.out.println(output.trim());
                output = "";
            }
            
            lastKey = key;
            output += word + " ";
        }
		
        if (!output.isEmpty()) {
        	System.out.println(output.trim());
        }
    }
}
```

squash 단계에서는 정렬된 파일을 읽어 같은 표시를 가지는 단어들을 한 줄로 합쳐 출력한다.

```shell
# 컴파일
javac Sign.java Sort.java Squash.java

# 파이프라인 실행
cat words.txt | java Sign | java Sort | java Squash
```
